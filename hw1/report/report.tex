\title{Building a Three-Layer Neural Network Classifier from Scratch: An Empirical Study on EuroSAT Land Cover Classification}

\author{Xiang Zheng, 25210980171}

\renewcommand{\shortauthors}{Xiang Zheng}

\begin{abstract}
This paper presents a from-scratch implementation of a three-layer fully connected neural network (MLP) for land cover classification on the EuroSAT satellite imagery dataset, using only NumPy for numerical computations. We investigate the foundations of backpropagation, gradient descent optimization, and regularization techniques, providing mathematical derivations and algorithmic implementations. Through empirical studies, we examine the impact of activation functions, network architecture, learning rate scheduling, and regularization strength on classification performance. Our experiments show that careful hyperparameter tuning yields notable performance improvements, with the best configuration achieving \textbf{66.40\%} test accuracy. We also visualize learned first-layer weight patterns, finding that the network develops color-channel-biased and coarse spatial filters that capture global intensity distributions rather than localized features. Error analysis identifies systematic confusions between visually similar land cover classes, illustrating the limitations of fully connected architectures for spatial pattern recognition. This work serves as both an educational resource for understanding neural network fundamentals and an empirical study on the applicability of MLPs to remote sensing image classification.
\end{abstract}

\keywords{neural network, backpropagation, land cover classification, EuroSAT, hyperparameter optimization, weight visualization}

\subsection{Motivation and Objectives}

Deep learning has advanced computer vision and remote sensing applications, with modern architectures achieving strong performance on complex classification tasks. However, the widespread adoption of high-level frameworks such as PyTorch and TensorFlow has created a gap between practitioners and the fundamental mathematical principles underlying neural networks. This work addresses this gap by implementing a complete neural network pipeline from first principles, enabling a deeper understanding of the computational mechanisms that drive learning in artificial neural systems.

The primary objectives of this work are threefold: (1) to develop a rigorous implementation of forward propagation, backpropagation, and optimization algorithms using only basic matrix operations; (2) to systematically evaluate the impact of various hyperparameters on model performance through controlled experiments; and (3) to analyze the learned representations through weight visualization and error analysis, providing insights into what features the network learns to discriminate among land cover types.

\subsection{Problem Formulation}

We address the problem of land cover classification using satellite imagery from the EuroSAT dataset. Given an input image $\mathbf{x} \in \mathbb{R}^{H \times W \times C}$ where $H=W=64$ and $C=3$, the objective is to learn a mapping $f_\theta: \mathbb{R}^{12288} \rightarrow \{0, 1, \ldots, 9\}$ that assigns each image to one of ten land cover categories. The function $f_\theta$ is parameterized by a three-layer fully connected neural network with parameters $\theta = \{\mathbf{W}^{(l)}, \mathbf{b}^{(l)}\}_{l=1}^{3}$.

\subsection{Contributions}

The main contributions of this work include:
\begin{itemize}
    \item A complete, modular implementation of a three-layer MLP with manual backpropagation, supporting multiple activation functions, optimization strategies, and regularization techniques.
    \item An empirical study examining four hyperparameter dimensions: activation function selection, hidden layer capacity, learning rate scheduling, and regularization strength.
    \item Visualization and analysis of learned weight patterns, revealing coarse color and spatial patterns captured by the network.
    \item Error analysis identifying failure modes and providing insights into the limitations of spatial-agnostic architectures.
\end{itemize}

\subsection{Data Preprocessing Pipeline}

The preprocessing pipeline consists of four stages: (1) \textit{loading}, where images are read from disk in JPEG format; (2) \textit{normalization}, where pixel intensities are scaled from $[0, 255]$ to $[0, 1]$ to improve numerical stability during optimization; (3) \textit{flattening}, where the 3D image tensor is reshaped into a 1D vector of dimension $D = 64 \times 64 \times 3 = 12,288$; and (4) \textit{splitting}, where the dataset is partitioned into training (70\%), validation (15\%), and test (15\%) sets using stratified sampling to preserve class proportions.

The flattening operation, while necessary for fully connected architectures, represents a significant information bottleneck as it discards spatial relationships between pixels. This limitation motivates our subsequent analysis of learned representations and error patterns.

\section{Methodology}

\subsection{Network Architecture}

We employ a three-layer fully connected neural network with the following computational graph:
\begin{align}
\mathbf{z}^{(1)} &= \mathbf{W}^{(1)} \mathbf{x} + \mathbf{b}^{(1)}, \quad \mathbf{a}^{(1)} = \sigma(\mathbf{z}^{(1)}) \\
\mathbf{z}^{(2)} &= \mathbf{W}^{(2)} \mathbf{a}^{(1)} + \mathbf{b}^{(2)}, \quad \mathbf{a}^{(2)} = \sigma(\mathbf{z}^{(2)}) \\
\mathbf{z}^{(3)} &= \mathbf{W}^{(3)} \mathbf{a}^{(2)} + \mathbf{b}^{(3)}, \quad \hat{\mathbf{y}} = \text{softmax}(\mathbf{z}^{(3)})
\end{align}

where $\mathbf{W}^{(1)} \in \mathbb{R}^{D \times H}$, $\mathbf{W}^{(2)} \in \mathbb{R}^{H \times H}$, $\mathbf{W}^{(3)} \in \mathbb{R}^{H \times C}$ are weight matrices, $\mathbf{b}^{(l)}$ are bias vectors, $H$ denotes the hidden layer dimension, $C=10$ is the number of classes, and $\sigma(\cdot)$ represents an element-wise activation function.

\subsection{Activation Functions}

We implement three activation functions, each with distinct properties that influence learning dynamics:

\textbf{ReLU} (Rectified Linear Unit): $\sigma(x) = \max(0, x)$, with derivative $\sigma'(x) = \mathbb{I}(x > 0)$. ReLU introduces sparse activations and mitigates the vanishing gradient problem, but suffers from ``dying neurons'' when inputs are consistently negative.

\textbf{Sigmoid}: $\sigma(x) = (1 + e^{-x})^{-1}$, with derivative $\sigma'(x) = \sigma(x)(1 - \sigma(x))$. Sigmoid squashes outputs to $(0, 1)$, providing smooth gradients but susceptible to saturation in deep networks.

\textbf{Tanh}: $\sigma(x) = (e^x - e^{-x})/(e^x + e^{-x})$, with derivative $\sigma'(x) = 1 - \tanh^2(x)$. Tanh provides zero-centered outputs in $(-1, 1)$, often leading to faster convergence than Sigmoid.

\subsection{Loss Function and Regularization}

We employ the cross-entropy loss augmented with L2 regularization:
\begin{equation}
\mathcal{L}(\theta) = -\frac{1}{N}\sum_{i=1}^{N}\sum_{c=1}^{C} y_{ic} \log(\hat{y}_{ic}) + \frac{\lambda}{2}\sum_{l=1}^{3} \|\mathbf{W}^{(l)}\|_F^2
\end{equation}

where $N$ denotes batch size, $y_{ic} \in \{0, 1\}$ is the ground-truth label, and $\lambda$ controls regularization strength. The regularization term penalizes large weights, promoting smoother decision boundaries and reducing overfitting.

\subsection{Backpropagation Derivation}

The gradients are computed via backpropagation, applying the chain rule in reverse topological order. For the output layer, combining softmax and cross-entropy yields a simplified gradient:
\begin{equation}
\frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(3)}} = \hat{\mathbf{y}} - \mathbf{y}
\end{equation}

For hidden layers, gradients propagate backward through weights and activation functions:
\begin{align}
\frac{\partial \mathcal{L}}{\partial \mathbf{W}^{(l)}} &= (\mathbf{a}^{(l-1)})^\top \boldsymbol{\delta}^{(l)} + \lambda \mathbf{W}^{(l)} \\
\boldsymbol{\delta}^{(l-1)} &= \boldsymbol{\delta}^{(l)} (\mathbf{W}^{(l)})^\top \odot \sigma'(\mathbf{z}^{(l-1)})
\end{align}

where $\boldsymbol{\delta}^{(l)} = \partial \mathcal{L} / \partial \mathbf{z}^{(l)}$ and $\odot$ denotes element-wise multiplication.

\subsection{Optimization Strategy}

We employ Stochastic Gradient Descent (SGD) with momentum:
\begin{equation}
\mathbf{v}_t = \beta \mathbf{v}_{t-1} - \eta \nabla_\theta \mathcal{L}, \quad \theta_t = \theta_{t-1} + \mathbf{v}_t
\end{equation}

where $\eta$ is the learning rate and $\beta=0.9$ is the momentum coefficient. To prevent gradient explosion in the high-dimensional input space ($D=12,288$), we apply gradient clipping:
\begin{equation}
\nabla_\theta \mathcal{L} \leftarrow \nabla_\theta \mathcal{L} \cdot \min\left(1, \frac{c}{\|\nabla_\theta \mathcal{L}\|_2}\right)
\end{equation}

with clipping threshold $c=5.0$.

Learning rate decay follows a linear schedule:
\begin{equation}
\eta_t = \eta_0 \left(1 - \frac{t}{T}\right) + \eta_{\min}\frac{t}{T}
\end{equation}

where $T$ is the total number of epochs and $\eta_{\min} = 10^{-6}$.

\subsection{Weight Initialization}

Proper initialization is critical for training stability. We employ He initialization for ReLU networks and Xavier initialization for Sigmoid/Tanh:
\begin{equation}
W_{ij} \sim \mathcal{N}\left(0, \sqrt{\frac{\alpha}{n_{\text{in}}}}\right), \quad \alpha = \begin{cases} 2 & \text{(He)} \\ 1 & \text{(Xavier)} \end{cases}
\end{equation}

Additionally, we apply a conservative scaling factor of $0.5$ to the first and third layers to account for the extreme input/output dimension ratios.

\section{Experimental Design}

\subsection{Experimental Protocol}

All experiments follow a controlled protocol: (1) fix random seed to 42 for reproducibility; (2) train for 80 epochs with batch size 64; (3) apply linear learning rate decay; (4) save the model achieving highest validation accuracy; (5) report test accuracy using the best model. We conduct four experimental studies, each varying one hyperparameter while holding others constant.

\subsection{Experiment 1: Activation Function Analysis}

\textbf{Hypothesis}: Different activation functions induce distinct learning dynamics due to their gradient properties and output distributions.

\textbf{Protocol}: Hidden Size=256, Learning Rate=0.01, Weight Decay=0.0001.

\textbf{Analysis}: The experimental results reveal several important insights about activation function selection for satellite image classification.

\textit{Why Sigmoid outperforms ReLU}: Although ReLU is widely preferred in modern deep learning for its computational efficiency and sparse activations, Sigmoid achieves the best performance (62.81\%) on this task. Several factors may explain this: (1) the input pixel values are normalized to $[0, 1]$, which naturally matches Sigmoid's output range, creating a consistent signal distribution across layers; (2) Sigmoid's smooth, bounded gradients prevent extreme activation values that can destabilize training in high-dimensional input spaces ($D=12{,}288$); (3) the satellite images contain smooth color transitions and textures, which may benefit from Sigmoid's gradual nonlinearity rather than ReLU's hard threshold at zero.

\textit{Why Tanh performs poorly}: Tanh achieves only 50.79\% accuracy, notably worse than both alternatives. A likely cause is the interaction between Tanh's zero-centered output range $(-1, 1)$ and the downstream layers: the alternating positive and negative activations can lead to less stable gradient flow in this shallow network with high-dimensional input. Additionally, Tanh's gradients vanish more severely for inputs with large magnitude, which can occur frequently in the high-dimensional first layer ($D=12{,}288$), slowing convergence within the 80-epoch budget.

\textit{Practical implications}: These results suggest that activation function selection should be empirically validated for each specific task rather than relying on general recommendations. The suitable choice depends on input data characteristics, network architecture, and initialization strategy.

\textbf{Analysis}: The relationship between hidden layer size and classification performance reveals the model's position on the bias-variance tradeoff curve.

\textit{Monotonic improvement}: Performance improves consistently as hidden size increases from 64 to 512, with test accuracy rising from 53.04\% to 62.44\%. This indicates the model is in the \textbf{underfitting regime}---the network lacks sufficient capacity to capture the complexity of the 10-class land cover classification problem. \textit{Diminishing returns}: The improvement from 256 to 512 dimensions is modest (60.62\% $\rightarrow$ 62.44\%), while the parameter count doubles from 6.4M to 12.8M. This suggests we are approaching the point of diminishing returns, where additional capacity provides marginal benefits but increases computational cost and risk of overfitting.

\textit{No overfitting observed}: The gap between validation and test accuracy remains small across all configurations (typically $<1\%$), indicating that weight decay regularization effectively prevents overfitting even at high capacity. Despite the high parameter-to-sample ratio (12.8M parameters for 18,900 training samples), the model does not overfit, which is consistent with the task's inherent complexity.

\textit{Practical recommendation}: For this task, hidden size 256 offers a good balance between performance (60.62\%) and efficiency (6.4M parameters). The additional 2\% gain from 512 dimensions may not justify the doubled computational cost in resource-constrained scenarios.

\textbf{Hypothesis}: Learning rate critically affects convergence speed and final performance; excessive rates cause instability while insufficient rates lead to slow convergence.

\textbf{Analysis}: The learning rate experiments reveal a strong sensitivity of model performance to this hyperparameter, with both optimization failure and suboptimal convergence observed.

\textit{Optimal learning rate (0.005)}: This configuration achieves the best test accuracy of 66.40\%, a 5.8 percentage-point improvement over the baseline (0.01). A moderately small learning rate allows more careful navigation of the loss landscape, which is important for high-dimensional input spaces ($D=12,288$) where gradients can have complex structure. The linear learning rate decay further refines the optimization in later epochs, helping the model settle into a better local minimum.

\textit{Failure at high rates (0.05, 0.1)}: At learning rates above 0.01, the model fails to learn, achieving only 11.11\% accuracy (near chance level for 10 classes). Large learning rates cause the optimizer to overshoot minima and enter unstable regions of the loss landscape. Even with gradient clipping (threshold 5.0), the large step sizes lead to oscillations and divergence, and the model converges to a degenerate solution.

\textit{Suboptimal convergence at low rate (0.001)}: While achieving competitive accuracy (61.63\%), the lower learning rate converges more slowly. This rate is too conservative, making small steps that require many more epochs to reach optimal performance. The model may still be improving at epoch 80, suggesting that longer training could close the gap with the optimal configuration.

\textit{Why 0.005 works best}: This learning rate strikes the right balance: it is large enough to make meaningful progress across the 80 epochs, but small enough to avoid overshooting minima. Combined with learning rate decay, it enables both efficient exploration early in training and precise convergence later.

This limitation motivates convolutional architectures that build in translation equivariance and local receptive fields.

\section{Discussion}

\subsection{Key Findings}

Our experiments yield several insights:

\textbf{Activation Function Selection}: While ReLU is the default choice in modern deep learning, Sigmoid outperforms it on this task. This result suggests that activation function selection should be empirically validated rather than assumed based on general practice.

\textbf{Learning Rate Sensitivity}: The sharp performance cliff at high learning rates underscores the importance of careful tuning. Gradient clipping provides robustness but cannot fully compensate for inappropriate learning rates.

\textbf{Learned Representations}: Despite random initialization and no architectural bias toward spatial processing, the network learns color-channel-biased weights and coarse spatial patterns that reflect global intensity distributions. These representations are limited to low-level color and brightness statistics, consistent with the absence of spatial inductive bias in the architecture.

\textbf{Spatial Information Loss}: The fundamental limitation of fully connected architectures for image data is the destruction of spatial relationships through flattening. This explains the systematic confusions between classes distinguished primarily by spatial patterns (Highway vs. River).

\subsection{Limitations and Future Work}

\textbf{Architecture}: Convolutional neural networks (CNNs) would preserve spatial structure and likely achieve superior performance. Future work should implement CNNs from scratch for comparison.

\textbf{Data Augmentation}: Augmentation techniques (rotation, flipping, color jittering) could improve generalization by increasing effective training set size.

\textbf{Optimization}: Advanced optimizers (Adam, RMSprop) and learning rate schedules (cosine annealing, warm restarts) may accelerate convergence and improve final performance.

\textbf{Multispectral Data}: The EuroSAT dataset includes 13 spectral bands from Sentinel-2; utilizing additional bands could improve discrimination between spectrally similar classes.

\section{Conclusion}

This paper presented a from-scratch implementation of a three-layer neural network for satellite image classification. Through systematic experimentation, we showed that careful hyperparameter tuning yields notable performance improvements, achieving 66.40\% test accuracy on the EuroSAT dataset. Weight visualization revealed that the network learns global color and intensity patterns as first-layer features, rather than localized spatial detectors. Error analysis identified systematic failure modes arising from the spatial-agnostic nature of fully connected architectures, highlighting the importance of spatial structure for image understanding tasks. This work serves as both an educational resource for understanding neural network fundamentals and an empirical study on MLP applicability to remote sensing.
